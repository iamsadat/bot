01 · Overview

One team of agents.
Every decision, inspectable.

An orchestrator plans the hunt and coordinates eight specialist agents end to end — discovery, vetting, resume tailoring, peer critique, submission, tracking, and improvement. Every agent decision is recorded as a structured ReasoningTrace you can open and read, and nothing irreversible happens without your one-click approval. The whole system is covered by 401 passing tests.

02 · How every agent thinks

Deliberate, then act.
Never a black box.

Every agent in JobHunt runs the same disciplined loop: deliberate on the situation, act on a plan, critique its own output, then decide whether to ship or refine again. Self-refinement is bounded — it stops once a quality threshold is met or a retry cap is hit — so reasoning stays fast, explainable, and never spins forever.

① Deliberate ② Act ③ Critique ④ Decide

If the critique score falls short of the quality threshold, the loop refines and tries again — up to a bounded number of attempts — before the decision is recorded either way. The full trace, including rejected attempts, stays attached to the job.

One plan. Seven specialists.

The Orchestrator builds an execution graph for the hunt, then hands work to seven specialist agents in sequence — Discovery, Vetting, Resume (Architect), Peer Critique, Submission, Tracking, and Improvement — passing structured state forward at every handoff. 04 · Discovery

Real boards. Polite by default.

Discovery uses pluggable JobSource adapters to pull straight from real ATS boards — Greenhouse, Lever, Ashby, and Indeed via RSS — plus an offline FixtureSource for deterministic testing. A token-bucket rate limiter keeps every adapter polite to upstream APIs, and local filters for salary, location, remote preference, and a company veto list gate roles out before they ever reach Vetting.

GreenhouseLeverAshby Indeed (RSS)FixtureSource (offline)

Token-bucket rate limiting → local filters (salary · location · remote · veto-company) → only roles that pass continue to Vetting.

05 · Vetting

An explainable scorecard,
not a black-box score.

Every role gets a transparent scorecard across compensation, freshness, specificity, and location-fit, plus a ghost-job risk score. Pluggable enrichers — Glassdoor, Crunchbase, News, and a layoffs heuristic — feed a weighted score you can tune yourself; the layoff-risk signal automatically inverts so a risky signal lowers, not raises, the final score.

CompensationFreshness SpecificityLocation-fitGhost-risk

Enrichers: Glassdoor · Crunchbase · News · Layoffs heuristic — each contributes a tunable weight; layoff risk auto-inverts before the final blend.

06 · Resume Architect

Every bullet, evidence-backed.

The Resume agent tailors a resume and cover letter to each specific role. Every bullet it writes must cite a real experience id from your history — no hallucinated accomplishments, ever. It extracts keywords straight from the job description and reports a coverage percentage so you can see exactly how well the tailored draft maps to what the role is asking for.

0% coverage
JD keyword extractionEvidence-cited bullets Coverage %Cover letter

Each generated bullet links back to a specific entry in your real experience history — if it can't be cited, it doesn't ship.

07 · Peer critique

A second agent checks the work.

Before anything reaches you for approval, a separate Peer-Critique agent cross-checks the tailored resume for tone and evidence integrity, scoring culture-alignment with cosine similarity against the company's signals. It returns a clear ship / no-ship verdict — a second pair of eyes on every application, every time.

Tone checkEvidence check Culture-alignment (cosine)

A no-ship verdict sends the draft back to Resume Architect for another pass — the loop is bounded, just like every other agent's self-refinement.

08 · Human approval

Nothing ships without you.

However confident the agents are, nothing is ever submitted without your explicit, one-click approve or reject. Approve, and the job auto-advances straight to Applied on the Kanban — no extra steps, no second confirmation needed. Reject, and it's archived with the reasoning trace intact for later review.

09 · Submission & autofill

Real auto-submit where it's safe.

For Greenhouse, JobHunt submits a real multipart application; for Lever, a real JSON payload — both gated behind your approval. For ATS platforms without a public API — Workday, iCIMS, and generic forms — a driver-agnostic autofill engine maps your profile fields onto the form instead. In every case, EEO and demographic questions are always deferred to you, never answered automatically.

Greenhouse — multipart API Lever — JSON API Workday / iCIMS / generic — autofill

Autofill is driver-agnostic: it maps your stored profile to whatever fields the form exposes, and always stops to ask before EEO / demographic questions.

10 · Tracking

Your inbox, read for you.

Tracking watches your inbox over IMAP today, with Gmail and Calendar API integration underway, and classifies replies — reject, interview, assessment, or offer — with a confidence score. It extracts Calendly and Zoom links and time hints from the message body, and the Kanban board advances itself the moment a reply lands. No manual status updates.

IMAP (today)Gmail / Calendar API (in progress) Calendly / Zoom extraction

Classification confidence is shown alongside each card — low-confidence reads stay flagged for a quick human glance.

11 · Continuous improvement

The strategy tunes itself.

A built-in A/B experiment framework runs deterministic assignment, promotes winners, and rolls back automatically when a variant underperforms. It's wired into a meta-agent that watches outcomes across the whole pipeline and tunes strategy over time — which resume phrasing wins more replies, which vetting weights find better roles — without you touching a config file.

Deterministic assignmentWinner promotionAutomatic rollback

The meta-agent observes every experiment's outcome and feeds the result back into how future strategies are chosen.

12 · Optional real LLM

Offline by default.
Sharper with a key.

JobHunt runs fully offline out of the box. Add an Anthropic API key and an optional, lazily-imported SDK integration switches on for tone rewrites and critique passes — the core pipeline never depends on it. Whenever it is used, personally identifiable information is redacted before anything leaves the process.

Lazy importFully offline without it PII redaction before send

Tone rewrites and peer critique get sharper with a real model in the loop — but every other capability in this tour works identically without one.

13 · Built to operate

Production plumbing, not a prototype.

Structured JSON logging with PII redaction, GitHub Actions CI with a pytest hard-gate, this very static site deployed to GitHub Pages, a Dockerfile and render.yaml for one-click backend hosting, and SQLite or Postgres persistence with an optional pgvector extension for semantic search. This is a system designed to run, not just to demo.
